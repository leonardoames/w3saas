

## Plan: Fix Shopee ADS Sync ‚Äî Complete Metrics and Date Range

### Problem Analysis

The `sync-shopee-ads` edge function currently:
1. **Only stores `investimento_trafego` (cost)** ‚Äî ignores clicks, impressions, conversions, and revenue from the Shopee Marketing API response
2. **Only fetches 30 days** ‚Äî should match the 90-day window used by the sales sync
3. **Sessions are empty** ‚Äî Shopee ADS "clicks" should map to `sessoes` (ad clicks = traffic sessions generated by ads)
4. **ROAS can't be calculated** ‚Äî because `faturamento` (revenue from ads conversions) is never stored

The Shopee Marketing API `get_campaign_daily_performance` returns fields like `cost`, `clicks`, `impressions`, `broad_gmv` (total revenue attributed to ads), `broad_order_num` (orders attributed), etc. The current code only reads `day.cost` and discards everything else.

### Changes

**1. Update `supabase/functions/sync-shopee-ads/index.ts`**

- Extend date range from 30 ‚Üí 90 days
- Extract additional fields from daily performance response:
  - `clicks` ‚Üí `sessoes`
  - `broad_gmv` (or `direct_gmv`) ‚Üí `faturamento` and `vendas_valor` (revenue attributed to ads)
  - `broad_order_num` (or `direct_order_num`) ‚Üí `vendas_quantidade`
  - `cost` ‚Üí `investimento_trafego` (already done)
- Store all these fields in the `metrics_diarias` upsert, not just `investimento_trafego`
- Use local date components in `formatDate` to avoid timezone shifts
- Add rate-limit delays between campaign fetches (already has 200ms, increase if needed)

**2. Data mapping**

```text
Shopee API field        ‚Üí  metrics_diarias column
‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ
cost / 100000           ‚Üí  investimento_trafego
clicks                  ‚Üí  sessoes

```

This allows the Dashboard to compute ROAS = faturamento / investimento_trafego for the `shopee_ads` platform filter.
